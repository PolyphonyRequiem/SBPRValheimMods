## 1. Install on your client (one-time per build)

1. BepInExPack_Valheim installed (r2modman or manual).
2. Copy `build/SBPR.Trailborne-0.1.0.zip` contents to client `BepInEx/plugins/SBPR.Trailborne/`. Folder must contain: `SBPR.Trailborne.dll`, `trailblazers_spade_v0.1.png`, `ink_*.png` (4), `cairn_marker_v0.1.png`.
3. Launch Valheim. BepInEx console should log:
   ```
   [Info   :   BepInEx] Loading [SBPR Trailborne 0.1.0]
   [Info   :SBPR Trailborne] [Trailborne] Awake — SBPR Trailborne 0.1.0 booting
   [Info   :SBPR Trailborne] [Trailborne] Harmony patches applied.
   ```

## 3. Acceptance checklist

### M0 — base infrastructure

- [ ] Hammer build menu Crafting tab has **Explorer's Bench** (10 Wood + 5 Stone, placeholder = vanilla workbench mesh)
- [ ] Place it; standing in range opens crafting UI showing the bench as the station
- [ ] Bench Furniture tab shows **Path Lamp** (3 ElderBark + 2 Resin) — places as a vanilla wood torch
- [ ] Hammer Misc tab has the spade-cycle Pieces — see M3 below

### M1 — inks + painted signs

- [ ] **Inks** craftable at Explorer's Bench: Red Ink (1 Raspberry → 2), White Ink (1 BoneFragments → 2), Blue Ink (1 Blueberries → 2), Black Ink (1 Coal → 2). Stackable 50, weight 0.1.
- [ ] Hammer Furniture tab shows **Painted Sign (Red/White/Blue/Black)** — each 2 Wood + 1 matching ink
- [ ] Placed sign is visibly tinted in its color (vanilla sign mesh + color overlay)
- [ ] Press **E** on a placed sign: vanilla text-edit dialog opens — write something
- [ ] Press **Shift+E** on the same sign: HUD shows `Pinned: <your text>` and a colored pin appears on the map (or minimap) at the sign's location
- [ ] Pin persists across server reconnect

### M2 — cairns + comfort

- [ ] **Cairn Marker** craftable at Explorer's Bench (2 Wood + 1 White Ink)
- [ ] Hammer Crafting tab has **Cairn** (9 Stone + 1 Cairn Marker). Placeholder mesh = vanilla bonfire.
- [ ] Place a Cairn near your base. Build a shelter (4 walls + roof) over a campfire.
- [ ] Stand inside the shelter near the Cairn: comfort level shown in Rested buff should be **at least 3** (Cairn floor). With a fire it should be 4+. Open inventory → hover Rested icon to read.
- [ ] **Cairn floor only applies when sheltered** — step outside, comfort drops back to vanilla numbers.

### M3 — Trailblazer's Spade real path ops

- [ ] Craft **Trailblazer's Spade** at Explorer's Bench (5 Wood + 5 Stone). Placeholder mesh = vanilla Hoe.
- [ ] Equip spade. Vanilla Hoe Piece menu opens (TWEAK ME: spade-only menu is M3.5 cleanup) — scroll wheel cycles.
- [ ] You'll see vanilla ops (raise, level, paved_road) PLUS our 4 new ops:
  - `Spade: Path (1.5m)`
  - `Spade: Path (3m)`
  - `Spade: Path (5m)`
  - `Spade: Replant (5m)`
- [ ] Select `Spade: Path (5m)` — left-click on ground; should paint a 5m-radius path tile (wider than vanilla 2m path).
- [ ] Select `Spade: Replant (5m)` — left-click on a cleared/dirt area; vanilla cultivate behavior at 5m radius (grass restored).

## 4. v0.1.0 limitations (NOT bugs)

- Placeholder models throughout — vanilla mesh + material tints. Bespoke art = v0.2+.
- Cairn = bonfire mesh (tier-1 ONLY; tier 2-5 upgrade + repair + downgrade + collapse all deferred to v0.2.0).
- Spade still shows vanilla Hoe ops because we share the Hoe's PieceTable (M3.5 cleanup carves out a spade-only table).
- ClearVegetation (the third spade mode in PARKED) — deferred. v0.1.0 ships only Path/Replant. Cultivate-replant covers the "restore terrain" use case.
- **Map fog default** is enforced by Niflheim's `-preset hardcore` modifier (no map at all). Trailborne does NOT yet patch Minimap on non-hardcore servers — that's v0.2.0 work to make the mod stand on its own.
- **Vanilla clients cannot join Niflheim while Trailborne is loaded** — `OnSBServer => true` stub will prefab-mismatch them. Real client/server handshake is M5/v0.2.0.
- **No localization file** — display names are baked English strings. JSON locale support = v0.2.0.

## 5. Hard-fail diagnostic procedure

If something doesn't behave as listed above:

1. Grab `~/AppData/Roaming/r2modmanPlus-local/Valheim/profiles/<profile>/BepInEx/LogOutput.log` from your client (or wherever your BepInEx logs land)
2. Grab the server tail: `sg docker -c "docker logs niflheim-server --tail 200"` from RequiemSoul
3. Drop both at me with the symptom
