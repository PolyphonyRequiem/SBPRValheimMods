## 1. Install on your client (one-time per build)

1. BepInExPack_Valheim installed (r2modman or manual).
2. Copy `build/SBPR.Trailborne-0.1.0.zip` contents to client `BepInEx/plugins/SBPR.Trailborne/`. Folder must contain: `SBPR.Trailborne.dll`, `trailblazers_spade_v0.1.png`, `ink_*.png` (4), `cairn_marker_v0.1.png`.
3. Launch Valheim. BepInEx console should log:
   ```
   [Info   :   BepInEx] Loading [SBPR Trailborne 0.1.0]
   [Info   :SBPR Trailborne] [Trailborne] Awake — SBPR Trailborne 0.1.0 booting
   [Info   :SBPR Trailborne] [Trailborne] Harmony patches applied.
   ```

## 3. Acceptance checklist

### M0 — base infrastructure

- [ ] Hammer build menu Crafting tab has **Explorer's Bench** (10 Wood + 5 Stone, placeholder = vanilla workbench mesh)
- [ ] Place it; standing in range opens crafting UI showing the bench as the station
- [ ] Bench Furniture tab shows **Path Lamp** (3 Wood + 2 Resin) — places as a vanilla wood torch
- [ ] Hammer Misc tab has the spade-cycle Pieces — see M3 below

### M1 — inks + painted sign

- [ ] **Inks** craftable at Explorer's Bench: Red Ink (1 Raspberry → 2), White Ink (1 BoneFragments → 2), Blue Ink (1 Blueberries → 2), Black Ink (1 Coal → 2). Stackable 20, weight 0.1.
- [ ] Hammer Furniture tab shows **exactly ONE Painted Sign** — `2 Wood` (no ink in the build cost; no Red/White/Blue/Black variants)
- [ ] Placed sign is **unpainted** (plain vanilla wood, no color tint)
- [ ] **Paint it:** with an ink in hand/hotbar, apply the ink to the placed sign → HUD shows `Painted sign <color>.` and the sign visibly tints to that color, consuming one ink
- [ ] **Repaint:** apply a *different* ink → sign re-tints to the new color (one ink consumed)
- [ ] Applying the **same** color again → HUD `Sign is already painted <color>.`, no ink consumed
- [ ] Painted color **persists across server reconnect** (and is visible to other joined clients)
- [ ] Press **E** on a placed sign: vanilla text-edit dialog opens — write something
- [ ] (Deferred) Shift+E map-pin gesture is **not** wired yet in v0.1.0 — no pin appears. This is expected; the pin path is a tracked follow-up.

### M2 — cairns + comfort (FULL 5-TIER v0.1.0)

- [ ] **Cairn Marker (Red/White/Blue/Black)** craftable at Explorer's Bench: 2 LeatherScraps + 1 FineWood + 1 matching Ink. Color you craft = color the placed cairn takes.
- [ ] Hammer Crafting tab shows **Cairn (Red/White/Blue/Black)** — Tier-1 build cost: **9 Stone + 1 Resin + 1 matching Cairn Marker**.
- [ ] Place a Cairn near your base. Build a shelter (4 walls + roof) over a campfire.
- [ ] Stand inside the shelter near the Cairn: comfort level shown in Rested buff should be **at least 3** (Cairn Tier-1 floor). With a fire it should be 4+. Open inventory → hover Rested icon to read.
- [ ] **Cairn floor only applies when sheltered** — step outside, comfort drops back to vanilla numbers.
- [ ] **Combo gesture — E-press on cairn**:
  - Pristine (≥75% HP) → hover text reads "Pristine — no maintenance needed", press is a no-op
  - Fizzled (<75% HP) + tier 1-4 → cost **3 Stone + 1 Resin**, repairs to max **and** upgrades to next tier (HUD: "Repaired + upgraded to T{n+1}")
  - Fizzled (<75% HP) + tier 5 → cost **3 Stone + 1 Resin**, repairs to max only (HUD: "Repaired")
  - Insufficient materials → HUD: "Need 3 Stone + 1 Resin"
- [ ] **Comfort floor scales with tier**: T1=3 / T2=4 / T3=5 / T4=6 / T5=7. After each upgrade, hover Rested icon and confirm the floor jumped.
- [ ] **Visual scales with tier**: T1 = small flattened-rock pile (4 stones); each tier adds ~2 more flattened rocks at progressively higher / smaller layers (T5 = 12 stones in a wider, taller stack). Same cairn looks the same after a relog (deterministic from ZDO seed).
- [ ] **Combat-immune**: hit the cairn with a sword / club / arrow. HP does NOT change. Same when a greyling smacks it — nothing happens.
- [ ] **Out-of-zone decay**: log out of the chunk for a few in-game days (sleep through several nights elsewhere or unload zone), come back. Hover HP — should have dropped at ~10 HP/in-game-day. Look in the server log for `Cairn backfill: missed Xd → decayed Y HP (...)`.
- [ ] **Shift+E debug damage (v0.1.0 only, default ON)**: with a pristine cairn (≥75% HP), press Shift+E. HUD: "(debug) Cairn damaged to ~70% HP". Confirm the hover text now shows the repair+upgrade affordance and combo-E completes the gesture cleanly. This flag is gated by `SBPR_DebugCairnDamage` in `BepInEx/config/net.danielgreen.sbpr.trailborne.cfg` — flip false (or delete the section) if you want to test pure natural decay.

(Regression guard for PR #16: widening the grass brush must NOT reintroduce terrain modification.) Cross-check the smallest against a real Cultivator's Grass mode — the effect should match, just at the spade's selected footprint.

## 4. v0.1.0 limitations (NOT bugs)

- Placeholder models throughout for benches/lamps/signs/spade — vanilla mesh + material tints. Bespoke art = v0.2+.
- **Cairns** use a runtime kitbash of flattened `rock_low` stacks per tier — not bespoke meshes yet, but they DO visibly scale with tier. Bespoke cairn art = v0.2+.
- Out-of-zone decay rate is a conservative 10 HP/in-game-day proxy. Real wear-curve tuning lands in v0.2.0.
- Resin-glow VFX on pristine cairns is **not** yet wired (decay state machine reports it via hover text only). Real glow effect = v0.2.0.
- Spade still shows vanilla Hoe ops because we share the Hoe's PieceTable (M3.5 cleanup carves out a spade-only table).
- ClearVegetation (the third spade mode in PARKED) — deferred. v0.1.0 ships only Path (×3) / Replant (×3). The Replant Grass ops cover the "restore cleared terrain" use case (vanilla Cultivator grass-replant behavior, now at 3 selectable widths).
- **Map fog default** is enforced by Niflheim's `-preset hardcore` modifier (no map at all). Trailborne does NOT yet patch Minimap on non-hardcore servers — that's v0.2.0 work to make the mod stand on its own.
- **Vanilla clients cannot join Niflheim while Trailborne is loaded** — `OnSBServer => true` stub will prefab-mismatch them. Real client/server handshake is M5/v0.2.0.
- **No localization file** — display names are baked English strings. JSON locale support = v0.2.0.
