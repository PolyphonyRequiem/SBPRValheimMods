# Research — docs/v3

Investigation, QA, and analysis write-ups supporting v3 (Swamp-tier) work. These
are evidence documents — what was measured, how, and the PASS/FAIL verdict —
distinct from the planning specs in `docs/v3/planning/` (which describe intended
behavior) and the design docs in `docs/design/` (which capture locked decisions).

A research doc records the *verification trail*: build hashes, probe methodology,
the verified-vs-reasoned split (logs-green ≠ playable), and what remains for a
human in-game check.
