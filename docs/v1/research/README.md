# v1 / research

Implementation investigations and fix notes — **point-in-time records** of how a
specific problem was understood and solved. These are `historical` by nature:
they capture a moment of reasoning, not a living spec. Don't update them to "stay
current"; if the design changes, the spec (`../../v0.1.0/planning/requirements.md`)
and the ADRs (`../../decisions/`) are where that lives.

See [`index.md`](index.md) for the file manifest.
